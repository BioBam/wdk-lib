# Workflow Development Kit

**wdk** is an open-source software development framework for defining Common Workflow Language (CWL) applications and reusable abstractions using familiar programming languages and rich object-oriented APIs. wdk apps synthesize into standard CWL workflows which can be sent to any WES service that accepts CWL.


## CWL Standard Specification

The code generated by this library follows the CWL standard specification v1.2

v1.3 draft1 is available at the link below, and previous versions are linked in the page.

https://www.commonwl.org/v1.3/Workflow.html



## Overview

**wdk** apps are programs written in one of the supported programming
languages. They are structured as a tree of
[constructs](https://github.com/aws/constructs).

The root of the tree is an `App` construct. Within an app, users define any
number of workflows (classes that extend the `Workflow` class). Each workflow is
synthesized into a separate Workflow cwl file. Workflows are, in turn,
composed of any number of constructs, and eventually from resources, which
represent any Workflow resource, such as `Tool`, `Container`, `Resource` etc.


wdk apps only ***define*** CWL applications, they don't actually run
them in a cluster. When an app is executed, it *synthesizes* all the workflows
defined within the app into the `dist` directory, and then those workflows can be
executed using a workflow runner like with the reference implementation cwl runner `cwltool dist/my_workflow.cwl my_inputs.yml` or a workflow management system like Toil, Arvados, CWL-Airflow, etc. The workflows can be published in workflow repositories while still in the main repo.



> **wdk** is based on the design concepts and technologies behind the [AWS
Cloud Development Kit](https://aws.amazon.com/cdk), ~~and can interoperate with
AWS CDK constructs to define cloud-native applications that include both
wdk resources and other CDK constructs as first class citizens.~~

## Tests

The library is thouroughly tested and the tests can be executed using the command:

```
# alias for: npx projen test
pj test

# run a specific .test.ts
pj test <.test.ts file path>
```

The test files come from multiple websites and projects that implement CWL workflows and tools, and try to reproduce those use cases.
